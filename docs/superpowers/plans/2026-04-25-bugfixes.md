## Task 1: Fix inverted interval trend detection in `_summarize_openf1_intervals`

**Background:** `get_intervals` (openf1.py) returns rows sorted ascending by date — `intervals[0]` is the earliest race moment, `intervals[-1]` is the most recent. The bug is that `_summarize_openf1_intervals` reverses the list and then names the variables backwards (`earliest_gap` gets the most-recent value, `latest_gap` gets the oldest), which inverts both the trend labels and the returned `latest_gap_to_leader*` keys.

**Files:**
- Modify: `server/f1_data.py:2174-2203`
- Test: `server/tests/test_f1_data.py`

def test_summarize_openf1_intervals_dropping_back():

```python
def _summarize_openf1_intervals(intervals: list[dict]) -> dict | None:
    if not intervals:
        return None

    def _parse_gap(value):
        if value is None:
            return None
        text = str(value).strip().replace("+", "")
        try:
            return float(text)
        except ValueError:
            return None

    # intervals arrive sorted ascending (earliest first) from get_intervals
    parsed = [_parse_gap(row.get("gap_to_leader")) for row in intervals]
    valid = [value for value in parsed if value is not None]
    if not valid:
        latest = intervals[-1]  # most recent entry
        return {
            "latest_gap_to_leader": latest.get("gap_to_leader"),
            "latest_interval": latest.get("interval"),
            "sample_count": len(intervals),
        }

    earliest_gap = valid[0]   # first non-None = chronologically earliest
    latest_gap = valid[-1]    # last non-None = chronologically most recent
    min_gap = min(valid)
    max_gap = max(valid)
    trend = "stable"
    if latest_gap > earliest_gap + 0.75:
        trend = "dropping_back"
    elif latest_gap < earliest_gap - 0.75:
        trend = "closing"

    return {
        "latest_gap_to_leader": intervals[-1].get("gap_to_leader"),  # most recent raw value
        "latest_interval": intervals[-1].get("interval"),
        "sample_count": len(intervals),
        "earliest_gap_to_leader_s": round(earliest_gap, 3),
        "latest_gap_to_leader_s": round(latest_gap, 3),
        "min_gap_to_leader_s": round(min_gap, 3),
        "max_gap_to_leader_s": round(max_gap, 3),
        "trend": trend,
    }
```

## Task 2: Fix stint `position_start`/`position_end` to use actual first/last lap position

**Background:** In `get_driver_strategy`, `positions` is built by iterating laps in stint order (`laps[0]` = first lap, `laps[-1]` = last lap). The code incorrectly uses `min(positions)` (best track position) for `position_start` and `max(positions)` (worst track position) for `position_end`. In F1, position 1 < position 20, so min/max produce best/worst, not first/last.

**Files:**
- Modify: `server/f1_data.py:814-815`
- Test: `server/tests/test_f1_data.py`

- [ ] **Step 1: Write the failing test**

Add to `server/tests/test_f1_data.py`:

## Task 3: Fix DNF list excluding lapped finishers in `get_race_report`
